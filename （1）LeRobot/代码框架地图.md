# LeRobot 代码框架地图

核心代码在 ==`（1）LeRobot下载/src/lerobot/`== 下。

## 一、顶层模块地图(src/lerobot/)

| 模块 | 管什么 / 看它学什么 |
|---|---|
| ==`scripts/`== | **所有命令行入口**(train/record/teleoperate...)。想知道"一条命令背后跑了啥",从这进 |
| ==`policies/`== | **策略模型实现**:ACT/Diffusion/VQ-BET/SmolVLA/π0 等各一个子文件夹。==看模型结构看这== |
| ==`datasets/`== | **LeRobotDataset 数据集格式**:怎么存、怎么读、视频怎么编码 |
| `robots/` | 各种**机械臂驱动/配置**(SO-ARM 等),发指令、读状态;==决定数据集存哪些特征== |
| `teleoperators/` | **遥操设备**(主动臂/手柄)接入 |
| `motors/` | **舵机底层通信**(feetech/dynamixel),`scservo_sdk` 在这层 |
| `cameras/` | **相机抽象**(OpenCV/RealSense),`find_cameras` 用的就是这 |
| ==`configs/`== | **配置 dataclass 定义**。想知道命令行能传哪些 `--参数`,看这 |
| `processor/` | 数据**预处理/后处理**管线(归一化、张量化、rename) |
| `optim/` | 优化器 / 学习率调度 |
| `envs/` | **仿真环境**接入(评估用) |
| `rl/` + `rewards/` | 强化学习(HIL-SERL 等)+ 奖励模型 |
| `transforms/` | 图像/数据增强 |
| `data_processing/` | 数据集转换、统计等加工 |
| `async_inference/` + `rollout/` + `transport/` | **部署/推理**:异步推理、真机执行、通信 |
| `model/` | 通用模型组件(backbone、网络层) |
| `utils/` `common/` `types.py` | 工具函数、通用代码、类型定义 |

> 根目录的 ==`examples/`==(不在 src 里)是官方教程脚本,==最友好的入口==,先跑通它。

## 二、policy 文件夹的"三件套"规范(认准一次,通吃所有模型)

LeRobot 借用 HuggingFace `transformers` 命名规范,==每个 policy 文件夹结构都一样==:

| 文件 | 是什么 / 看它干啥 |
|---|---|
| `__init__.py` | 包入口,导出类。基本不用看 |
| ==`configuration_<名>.py`== | ==超参配置(dataclass)==:维度、层数、chunk、归一化、输入输出特征。"模型怎么搭、参数填什么"。命令行 `--policy.xxx` 对应这里 |
| ==`modeling_<名>.py`== | ==模型本体==:PyTorch `nn.Module`,前向、loss、predict。==看网络结构/改网络代码看这== |
| `processor_<名>.py` | ==数据进出适配==:输入怎么归一化/组装,输出怎么反归一化 |
| `README.md` | 说明 / 论文 / 用法(有的指向 `docs/`) |

> 记法:==`configuration`=配置, `modeling`=网络, `processor`=数据进出==。

## 三、各模型一句话(policies/ 下共 17 个)

```
act  diffusion  eo1  gaussian_actor  groot  molmoact2  multi_task_dit
pi0  pi05  pi0_fast  rtc  smolvla  tdmpc  vla_jepa  vqbet  wall_x  xvla
```

| 文件夹 | 是什么 |
|---|---|
| ==`act`== | **ACT**(Action Chunking Transformer):分块预测动作,==双臂精细操作标配==(起步首选) |
| ==`diffusion`== | **Diffusion Policy**:扩散模型建模动作分布,擅长多模态 |
| `eo1` | **EO-1**:VLA,Qwen2.5-VL backbone + flow-matching 动作头,带语言指令 |
| `gaussian_actor` | **RL 用的高斯策略 actor**(输出动作高斯分布+Tanh),给 HIL-SERL 这类在线 RL 用;非完整策略,是构件 |
| ==`pi0` / `pi05` / `pi0_fast`== | **π0** 系列 VLA(Physical Intelligence) |
| ==`smolvla`== | **SmolVLA**:小型 VLA,==SO-ARM/消费级硬件友好== |
| ==`vqbet`== | **VQ-BET**:动作离散成 token 再预测 |
| ==`tdmpc`== | **TD-MPC**:基于模型的 RL |
| `groot` `molmoact2` `wall_x` `xvla` `vla_jepa` `multi_task_dit` `rtc` | 更新/更大的 VLA 或实验性模型,初期不用管 |

## 四、★ 想改什么 → 改哪个文件(速查表)

### 改模型结构
| 想改什么 | 改哪里 |
|---|---|
| 超参:`dim_model` / `n_heads` / `n_encoder_layers` / `chunk_size` / 视觉 backbone | ==`configuration_<名>.py`==(改字段或命令行 `--policy.xxx`,**不用动代码**) |
| 网络本身:加层、换注意力、改 loss、改 forward | ==`modeling_<名>.py`==(改 PyTorch 代码) |

> ACT 默认值(grep 自 `configuration_act.py`):`chunk_size=100`、`dim_model=512`、`n_heads=8`、`n_encoder_layers=4`、`n_decoder_layers=1`、`latent_dim=32`、`vision_backbone=resnet18`。

### 改数据结构(★ 不在 modeling 里!分清三种"数据")
| 你想改的"数据" | 真正改的地方 |
|---|---|
| 数据集**存什么**:几个相机、几个关节、动作多少维 | ==采集端==:`robots/` 机器人配置 + 录制 `lerobot_record.py`。写进数据集 `meta` features,**不在 policy 里** |
| 模型**接受哪些输入/输出特征** `input_features`/`output_features` | ==`configuration_<名>.py`==(通常自动推断,改映射/维度在这) |
| 喂模型前**怎么预处理**:归一化、图像增强、组装 | ==`processor_<名>.py`== + `transforms/` + config 的 `normalization_mapping` |
| 数据集**底层存/读格式** | `datasets/`(LeRobotDataset,一般**别动**) |

> [!important] 一句话总结
> - ==`modeling_<名>.py` 只管"网络怎么算",不管"数据长什么样"。==
> - 改**网络代码** → `modeling`;改**超参** → `configuration`;改**数据预处理** → `processor`;改**数据集存什么** → 采集端(`robots/` + `record`)。
